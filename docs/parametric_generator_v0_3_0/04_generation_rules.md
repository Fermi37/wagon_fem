# Generation Rules

## Shared Rules

The tank generator should follow the deterministic construction principles of `v0.1.0`:

- merge coordinates by tolerance;
- create nodes from sorted coordinate keys;
- create members in a stable group order;
- reject duplicate members for the same unordered node pair and member tag;
- export semantic `member_tag` and `section_tag` values.

## Generation Order

The recommended generation order is:

1. Normalize parameters and derived dimensions.
2. Build the global station list along `x`.
3. Build underframe nodes and members.
4. Build tank ring nodes at each station.
5. Build tank longitudinal members between adjacent stations.
6. Add tank end-ring and end-dish equivalent members.
7. Add saddle, lug, strap, and pad members.
8. Add optional equipment attachment members.
9. Assign loads and supports.
10. Validate graph, coordinates, load balance, and solver readiness.

## Station Rules

The station list must include:

- `geometry.end_positions`;
- `tank.x_start` and `tank.x_end`;
- `geometry.bolster_positions`;
- saddle positions;
- middle-lug positions;
- manhole and drain positions;
- regular ring stations generated by pitch;
- user-specified `tank.extra_ring_positions`.

The implementation should preserve each source in metadata so a generated station can be audited.

## Underframe Rules

Create the center sill as a longitudinal polyline through all frame stations. Create bolster beams at each bolster station from left support-side coordinate to right support-side coordinate. Create end beams at `x = 0` and `x = frame_length`.

Intermediate cross beams should be generated from `layout.cross_beam_pitch`, with mandatory insertion at tank support stations and equipment attachment stations.

Side beams are generated only when `frame.include_side_beams = true`. Console side beams should be limited to zones outside the tank middle region when the parameter file requests a lighter tank-car frame.

## Tank Ring Rules

At each tank station, generate a polygonal ring with `layout.angular_divisions` nodes. The angular grid starts at the right-side spring line, proceeds counterclockwise in the `y-z` plane, and stores angles in metadata.

Each adjacent angular pair becomes a `tank_ring` member. The closing member connects the last angular node to the first angular node.

Stations at `tank.x_start` and `tank.x_end` receive `tank_end_ring` tags. Optional end-dish spokes connect selected ring nodes to an end-dish reference node on the tank axis.

## Tank Longitudinal Rules

For each angular index, connect nodes with equal angular position between adjacent stations. The default tag is `tank_longitudinal`.

The top and bottom generators may receive additional tags:

- `tank_top_stringer` near `theta = 90 deg`;
- `tank_bottom_stringer` near `theta = 270 deg`.

These additional tags support postprocessing without changing the physical member count.

## Interface Rules

Saddle supports connect tank ring nodes within the specified angular span to frame nodes at the same station. The first-stage representation may use one vertical pad member at the bottom generator and two inclined pad members at adjacent angular generator nodes.

Middle lugs connect a bottom or lower-side tank node to the center sill. If `longitudinal_lock = true`, the generated metadata should mark this interface as the primary longitudinal connection.

Strap ties connect lower-side tank nodes to side or bolster nodes. The first implementation may represent each strap by two inclined beam members with `strap_tie` tags.

## Load Assignment Rules

Distributed tank self-weight and payload should be converted into nodal loads at tank lattice nodes or uniform loads on tank longitudinal members. The selected placement method must be recorded in metadata.

For force-flow studies, load assignment should support both:

- direct loads on tank bottom generators;
- sector-weighted loads on all tank lattice nodes below the fill level.

## Stable Identifier Rules

Identifier stability requires:

- stable sorted coordinate lines;
- stable angular grid;
- stable member-group order;
- deterministic traversal within each member group;
- explicit tie-breaking by `(x, y, z, member_tag)`.
